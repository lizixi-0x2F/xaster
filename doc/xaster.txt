1. INTRODUCTION                                              *xaster-intro*

Xaster is a Neovim-native AI coding agent. Unlike simple "chat with AI in a
sidebar" plugins, Xaster implements a full autonomous agent loop:

    EXPLORE → PLAN → EXECUTE → VERIFY

The agent reads your codebase, plans the work, makes edits through real Vim
operations, shows you every diff before applying, and verifies nothing broke
— all without leaving Neovim.

Key features:
  - True agent loop with phase-gated tool access
  - 70+ native tools including Vim grammar (ciw, da{, :%s/)
  - Edit confirmation with unified diff viewer
  - Virtual cursor showing agent position in real-time
  - Four-layer memory system (fast, slow, jump stack, snapshots)
  - Automatic LSP diagnostic verification after edits
  - Circuit breaker to prevent infinite loops
  - Checkpoint recovery across Neovim sessions
  - Bring-your-own-model: any OpenAI-compatible API

2. INSTALLATION                                            *xaster-install*

REQUIREMENTS ~
  - Neovim >= 0.10
  - `curl` executable (for API calls)

LAZY.NVIM ~
>
  {
    "lizixi-0x2F/xaster",
    opts = {
      agent = {
        api_key = "sk-...",
        model = "gpt-4o",
      },
    },
    keys = {
      { "<leader>ac", "<cmd>XasterChat<cr>", desc = "Xaster Chat" },
    },
  }
<

PACKER.NVIM ~
>
  use {
    "lizixi-0x2F/xaster",
    requires = { "nvim-lua/plenary.nvim" },
    config = function()
      require("xaster").setup({
        agent = { api_key = "sk-...", model = "gpt-4o" },
      })
    end,
  }
<

ENVIRONMENT VARIABLES ~
  ANTHROPIC_API_KEY       API key for Anthropic/OpenAI-compatible endpoints
  ANTHROPIC_BASE_URL      Custom API endpoint URL
  ANTHROPIC_MODEL         Model name override

agent.api_key                                *xaster-config-agent-api_key*
    API key for the LLM provider. Required if no env var is set.

    Model name. Supports any OpenAI-compatible model.
    Examples: "gpt-4o", "deepseek-chat", "claude-sonnet-4-20250514".

    Maximum agent loop iterations before forced stop (default: 1024).

agent.ultracode                              *xaster-config-agent-ultracode*
    Boolean. When true, agent uses exhaustive mode with deeper analysis,
    more thorough execution, and adversarial self-review.

5. AGENT LOOP                                           *xaster-agent-loop*

Xaster follows a structured four-phase engineering cycle:

  EXPLORE → PLAN → EXECUTE → VERIFY

PHASE DETAILS ~

EXPLORE ~
  - Read-only phase. No edits allowed.
  - Agent reads files, runs LSP queries, browses symbols.
  - Goal: understand the codebase and problem space.

PLAN ~
  - Read + memory write tools. Still no edits.
  - Agent creates tasks with dependencies, designs approach.
  - Auto-advances from EXPLORE when tasks are created.

EXECUTE ~
  - Full tool access. All 70+ tools available.
  - Agent makes changes via Vim-native operations.
  - Every edit is diffed and shown for confirmation.

VERIFY ~
  - Full tool access with verification-focused prompt.
  - Runs LSP diagnostics, checks for regressions.
  - Auto-advances from EXECUTE after edits are made.

AUTO-ADVANCEMENT ~
  - EXPLORE → PLAN: when memory_tasks_init is called
  - EXECUTE → VERIFY: when any edit tool is executed
  - VERIFY → EXPLORE: when verification passes, if tasks remain

CONFIRMATION ~
  Before edits are applied, a unified diff is shown in a floating window.
  Press:
    y       Accept edits, continue agent loop
    n / q   Reject edits, restore pre-edit snapshot
    <C-c>   Abort, restore snapshot and stop agent

7. MEMORY                                                *xaster-memory*

Xaster has a four-layer memory system:

FAST MEMORY (TTL-based scratchpad) ~
  For temporary storage during a single task. Keys have optional TTL.
  memory.remember(key, value, ttl_seconds)
  memory.recall(key)           -- returns value or nil
  memory.forget(key)

SLOW MEMORY (persistent, confidence-scored) ~
  For knowledge that should survive context compression. Each entry has
  a confidence score (0.0-1.0) that increases with repeated confirmation.
  memory.learn(type, key, value, tags, confidence)
  memory.know(key)             -- direct lookup
  memory.query(key, type, min_confidence)  -- fuzzy search
  memory.guide()               -- most relevant high-confidence entries

JUMP STACK (semantic navigation) ~
  Structured forward/back navigation with intent labels:
  memory.jump_push() / memory.jump_back() / memory.jump_forward()
  Automatically pushes current position when jumping.

SNAPSHOTS (full editor state) ~
  Captures buffer content, cursor, all named registers, and all named marks.
  memory.snapshot_create(name)    -- full rollback point
  memory.snapshot_restore(name)   -- restore editor state
  memory.snapshot_diff(name)      -- diff against snapshot

TASK TRACKING ~
  Tasks with dependency DAG support:
  memory.tasks_init(tasks)
  memory.task_update(id, status)
  memory.task_progress()          -- returns blocked / in-progress / done

8. UI                                                        *xaster-ui*

CHAT PANEL ~
  Opens as right-side floating window with three vertical sections:
    - Chat buffer (top)    — Markdown with Treesitter highlighting
    - Command buffer (mid) — Tool execution status (PENDING / OK / ERROR)
    - Input buffer (bot)   — User input line

DIFF CONFIRMATION ~
  Floating window showing unified diff with:
    - Green + lines (additions)
    - Red - lines (deletions)
    - Blue hunk headers (@@ -old +new @@)
    - File-by-file summary

VIRTUAL CURSOR ~
  Agent position indicator:
    - Amber |  = editing
    - Purple | = reading
    - Fading trails show where the agent has been

HIGHLIGHT GROUPS ~
  XasterAgent      - Agent messages (Mauve)
  XasterThink      - Reasoning text (Gray italic)
  XasterUser       - User messages (Green)
  XasterTool       - Tool call names (Amber)
  XasterToolSuccess - Successful tool results (Green)
  XasterToolPending - In-progress tool calls (Yellow)
  XasterError      - Error messages (Red)
  XasterDiffAdd    - Added lines (Green background)
  XasterDiffDel    - Removed lines (Red background)
  XasterDiffHunk   - Hunk headers (Blue)
  XasterCodeBlock  - Inline code in chat
  XasterInlineButton - Action buttons

STATUSLINE ~
  Use `require("xaster").statusline()` in your statusline config to show
  agent phase, round count, and token usage.

9. SUPPORTED MODELS                                       *xaster-models*

Any provider with an OpenAI-compatible chat completions endpoint:

  Provider    Model example          Notes
  ─────────   ────────────────────   ─────────────────────────────
  OpenAI      gpt-4o, gpt-4-turbo   Native tool calling
  Anthropic   claude-sonnet-4-...    Via compatible endpoint
  DeepSeek    deepseek-chat          Full support, reasoning_content
  Groq        llama-3.3-70b          Fast inference
  Ollama      qwen2.5-coder:14b      Local, private
  OpenRouter  (any model)            Unified API gateway

DeepSeek-specific handling:
  - Tool list is intersected with essential tools to avoid "request too
    large" API errors
  - reasoning_content is streamed and displayed as XasterThink (gray italic)
  - All schemas use additionalProperties: false (strict mode)

10. KEYMAPS                                               *xaster-keymaps*

Default keymaps:
  <leader>ac     :XasterChat              Toggle chat panel
  <leader>as     :XasterAgentStop         Stop running agent
  <leader>xo     :XasterObserve           Editor state snapshot
  <leader>xh     :XasterHistory           Operation history
  <leader>xa     Toggle action indicator  Agent status indicator

Chat panel keymaps:
  Input buffer:   <CR> submit, <Esc> stop-insert, <C-k> focus chat
  Chat buffer:    i focus input, <C-c> stop agent
  Command buffer: i focus input, <C-k> focus chat, <C-c> stop agent

Diff confirmation keymaps:
  y     Accept edits, continue loop
  n / q Reject edits, restore pre-edit snapshot
  <C-c> Abort, restore snapshot and stop agent

All keymaps are configurable via the `keymaps` option in setup().

12. TROUBLESHOOTING                                *xaster-troubleshooting*

API KEY NOT CONFIGURED ~
  Xaster checks for API key in this order:
    1. config.agent.api_key in setup()
    2. Environment variable ANTHROPIC_API_KEY
    3. Environment variable ANTHROPIC_AUTH_TOKEN
  Use :XasterStatus to verify the agent is configured.

"CURL NOT FOUND" WARNING ~
  Xaster uses curl for API requests. Install curl:
    macOS:  (built-in)
    Ubuntu: sudo apt-get install curl

NEOVIM VERSION WARNING ~
  Xaster requires Neovim >= 0.10 for vim.system() support.
  Check: nvim --version

DEEPSEEK "REQUEST TOO LARGE" ~
  Xaster automatically filters tools to a minimal set for DeepSeek models.
  If you still encounter this, try reducing agent.max_rounds or clearing
  conversation history with :XasterAgentClear.

AGENT NOT RESPONDING ~
  1. Check :XasterStatus — is running=yes?
  2. Check :XasterLog 50 — look for ERROR entries
  3. Try :XasterAgentStop then :XasterAgentClear and restart
  4. Check your API credit/rate limits

RESTORE FROM CRASH ~
  If Neovim crashed mid-session:
    1. Reopen Neovim — Xaster shows checkpoint notification
    2. Run :XasterCheckpoints to see available saves
    3. Run :XasterRestoreLast to restore the latest checkpoint
